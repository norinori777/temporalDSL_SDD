# Temporal DSL 概要設計書

## 目的

このリポジトリでは、Temporal DSL を理解しながら、SaaS サービスへワークフロー機能を組み込むことを目的とする。
まずは Temporal DSL の構造、ワークフローの組み立て方、アクション定義、検証ルール、永続化の関係を整理し、実装の前提を共有できる状態にする。

## 実装の考え方

- ワークフローはアクションの組み合わせで表現する。
- アクションの選択肢は DB に登録されたスキーマ定義と YAML 宣言から導出する。
- 画面から組み立てたワークフローは DB に保存する。
- Temporal 実行時にも同じ定義を使って validation を行う。
- アクションは 3 つのマイクロサービスとして分離する。

## 制約

- 開始ノードは 1 つだけにする。
- 終了ノードは 1 つだけにする。
- DFS で循環を検出する。
- 未接続ノードを検出する。
- 並列ノードはブランチ数を検証する。
- 条件分岐ノードは分岐数を検証する。

## データ管理

### アクションスキーマ

- アクションスキーマは事前に DB へ登録する。
- スキーマにはバージョンを持たせる。
- バージョン管理により、後方互換性を保てるようにする。

### YAML 宣言

- アクションへのリクエスト宣言は YAML 形式で DB に保存する。
- YAML 宣言は、画面上で選択可能なアクションの判定に使う。
- 同じ YAML 宣言は、Temporal からのリクエスト検証にも使う。

### ワークフロー定義

- 画面で組み立てたワークフローは DB に保存する。
- 保存された定義は再読込できるようにする。
- 定義にはスキーマバージョンを含め、後続の互換性チェックに利用する。

## 開発方針

- 実装中のやり取りは日本語で行う。
- コードには日本語コメントを付け、意図を読み取りやすくする。
- 実装結果の設計書は `/docs` 配下に残し、Temporal DSL の理解を再現可能にする。
