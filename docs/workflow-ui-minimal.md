# 最小ワークフロー画面

## 目的

Temporal DSL の構造を隠さず、まずは最小のワークフロー編集画面として可視化する。
この画面はドラッグ&ドロップの完成版ではなく、Temporal DSL の学習に必要な要素を一画面で把握することを優先する。

## 画面の役割

- 左側: DB 登録済みアクション候補の一覧
- 中央: ワークフローのノード列と接続の見え方
- 右側: 選択中ノードの詳細と保存状態

## 学習上の意図

- アクションが `actionCode` と `version` を持つことを見せる
- 開始ノード、分岐ノード、終了ノードの役割を見せる
- 保存前に検証ルールを確認できるようにする
- requestYaml の存在を画面上で意識できるようにする

## 今後の拡張候補

- ノードの追加・削除・再接続
- DB から読んだアクション候補の反映
- 検証結果のリアルタイム表示
- 保存 / 読込 ボタンの実装
